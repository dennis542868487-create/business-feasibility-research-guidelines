# Governance

## Maintainer

The initial maintainer is [@dennis542868487-create](https://github.com/dennis542868487-create). The maintainer controls the upstream repository, release tags, GitHub Releases, marketplace metadata, and any official plugin submission.

## Contribution model

Anyone may view, clone, install, fork, modify, and propose changes under the MIT License. Public contributors should work in their own fork and open a pull request. Direct write access is not granted by default.

Pull requests must explain the decision impact, preserve evidence and safety guardrails, and pass the repository's automated checks. The maintainer may request changes, merge, or decline a proposal. A declined pull request may remain in the contributor's fork under the license.

## Decision policy

Changes are evaluated for methodological correctness, dimensional consistency, reproducibility, privacy, security, maintainability, and usefulness across business types. Compatibility and safety take priority over feature count.

## Releases

Stable releases use semantic versions and immutable `v*` tags. Release archives are built from the tagged source, tested in CI, and published with SHA-256 checksums. Only the maintainer or a future delegated release manager may publish an upstream release.

## Future maintainers

Sustained contributors may be invited as collaborators after a record of high-quality review and implementation. Access can be removed when it is no longer needed or when project security requires it.
